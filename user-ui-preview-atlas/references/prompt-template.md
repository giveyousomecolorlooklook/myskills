# UI 预览图生成拆件汇总图提示词模板

## 组件分析表模板

在生成前先整理这张表，必要时用像素测量辅助估算。

```text
组件名：
- 在预览图中的存在形式：独立组件 / 父组件 / 子组件 / 状态描边
- 预估预览图区域：x=?, y=?, w=?, h=?
- 预估宽高比：?
- 是否生成到拆件图：是 / 否
- 浮雕：无 / 轻微 / 明显
- 外描边：无 / 颜色 / 粗细 / 材质
- 内描边：无 / 颜色 / 粗细 / 材质
- 填充材质：?
- 禁止误生成点：?
```

## 中文提示词骨架

```text
任务：根据唯一输入预览图生成 UI 拆件汇总图。

唯一参考图：只参考输入的这张预览图，不参考旧版，不根据记忆补充元素，不生成参考图中不存在的组件。

第一步：先在内部分析参考图，不要直接自由创作。
请识别参考图中真实存在的 UI 组件，并判断每个组件：
1. 是否独立存在；
2. 是父组件、子组件，还是状态描边；
3. 预估宽高比；
4. 是否有浮雕；
5. 是否有外描边；
6. 是否有内描边；
7. 填充材质；
8. 是否包含子控件。

关键规则：
不要把状态效果误生成成独立组件。
如果某个颜色描边只是另一个组件的选中状态，不要把它生成成独立按钮、独立长条或独立面板。
只能生成完整状态背景，或者不生成该状态组件。

画布要求：
画布尺寸：2048x2048。
背景颜色：纯 #00ff00 绿色。
背景必须完全平整、无阴影、无渐变、无纹理、无环境、无地面。
所有组件之间必须留出足够绿色间距，方便后续切图。
所有组件必须完整显示，不要重叠，不要超出画布。
不要在组件内部使用 #00ff00。

拆件规则：
1. 按不可拆分组件拆分；
2. 按钮是不可拆分组件；
3. 文字占位条是不可拆分组件；
4. 带描边的背景面板视为不可拆分组件；
5. 父面板生成空面板，不要包含子控件；
6. 重复组件只保留一个；
7. 不生成参考图中不存在的组件。

风格要求：
严格保持参考图的 UI 风格、材质、边框厚度和明暗关系。
不要添加人物、场景、额外图标、文字、数字、水印。

生成组件清单和布局：
下面每个组件都必须放在指定的拆件图框内，保持指定宽高比和视觉属性。
```

## 单个组件条目模板

```text
N. 组件名称
- 拆件图位置：x=?, y=?, w=?, h=?
- 宽高比：约 ?:1，必须尽量接近参考图
- 存在形式：独立组件 / 父组件 / 子组件 / 状态描边
- 内容：空父面板 / 完整按钮 / 文字占位条 / 其他
- 浮雕：无 / 轻微 / 明显
- 外描边：无 / 描述颜色、粗细、材质
- 内描边：无 / 描述颜色、粗细、材质
- 填充：描述材质
- 禁止：不要生成什么
```

## 状态描边写法示例

```text
选中列表行背景
- 拆件图位置：x=96, y=1360, w=760, h=182
- 宽高比：约 4.18:1，必须接近参考图中完整选中行区域
- 存在形式：完整列表行的选中状态，不是独立青色长条
- 内容：空行背景，不包含头像、文字条、小按钮
- 浮雕：只有列表行背景本身的轻微浮雕
- 外描边：细青色选中描边，不能变成厚青色边框
- 内描边：如参考图，没有则不要添加
- 填充：暗色纹理
- 禁止：不要生成单独的青色按钮、长条、标签或发光横条
```

## 最终负面约束

```text
禁止：旧版设计细节、参考图外元素、额外图标、人物、场景、环境光、投影、接触阴影、反射、水印、文字、数字、绿色像素进入组件内部、组件重叠、完整最终界面截图、把状态描边独立成按钮。
```
